# v8.6
*13 de mayo de 2026*

## Tareas que no se pierden

Hasta ahora, si recargabas la página o cerrabas la ventana mientras el agente estaba trabajando, al volver encontrabas una burbuja vacía sin forma de saber qué había pasado. A partir de esta versión, cada tarea del agente se guarda en tiempo real: el contenido se va almacenando cada pocos segundos, y el estado de la tarea (en progreso, completada, fallida) queda registrado de forma permanente. Si vuelves a abrir el chat, Vibes detecta si la tarea sigue activa y actualiza el contenido automáticamente, o te muestra directamente el resultado final si ya terminó. Nada se pierde, aunque cierres y te vayas a tomar un café.

## Errores que se explican solos

Cuando algo falla en la comunicación con la IA, ya no verás un bloque de JSON técnico incomprensible. Vibes ahora intercepta cada error, lo clasifica internamente y te muestra un mensaje claro en español explicando qué ha pasado. Pero la mejora no se queda ahí: dependiendo del tipo de error, la burbuja incluye **botones de acción directa** para resolver el problema sin salir del chat.

Si el error es algo que se puede solucionar reintentando (un timeout, un error puntual del servidor, un problema de red), verás un botón **Reintentar** que recupera tu último mensaje y lo prepara para reenviar. Si el error requiere que hagas algo distinto (cambiar de modelo, recargar créditos, revisar tu API key), los botones te llevan directamente a donde necesitas ir: a Ajustes, a la página de créditos de OpenRouter o a un chat nuevo. En todos los casos puedes desplegar la sección **Ver detalles** para consultar el error técnico original si necesitas diagnosticar algo en profundidad.

Antes de mostrar cualquier error, Vibes intenta reparar el problema de forma silenciosa. Si lo consigue, la conversación continúa sin interrupciones y sin que notes nada. Si el auto-repair no funciona, entonces sí se muestra el error con las opciones manuales.

| Situación | Qué hace Vibes |
|---|---|
| El servidor del agente se ha caído | Lo reinicia automáticamente y reintenta tu mensaje |
| La sesión del agente se ha perdido | Crea una sesión nueva y reintenta |
| El agente está ocupado con otra tarea | Aborta la tarea anterior, espera 2 segundos y reintenta |
| Límite de solicitudes alcanzado | Espera 5 segundos y reintenta |

### Errores irrecuperables

Estos errores requieren una acción del usuario para resolverse. El botón de reintentar no aparece porque enviar el mismo mensaje no solucionaría nada.

| Problema detectado | Mensaje | Acciones disponibles |
|---|---|---|
| Créditos agotados | Parece que se agotaron los créditos de IA de tu cuenta. | Recargar créditos, Cambiar modelo |
| API key inválida | Parece que hay un problema con tu clave API. Revísala en ajustes. | Abrir Ajustes |
| Modelo no disponible | Parece que el modelo seleccionado no está disponible. Prueba con otro. | Cambiar modelo |
| Chat demasiado largo | Parece que el chat es demasiado largo para el modelo. Abre un nuevo chat o cambia a un modelo con mayor ventana de contexto. | Nuevo chat, Cambiar modelo |
| Contenido bloqueado | Parece que el contenido fue bloqueado por los filtros de seguridad del modelo. | — |
| Agente no encontrado | Parece que no se encontró el agente de IA. Reinicia Vibes para resolverlo. | — |
| Disco lleno | Parece que no queda espacio en disco. Libera espacio e inténtalo de nuevo. | — |

### Errores recuperables

Estos errores suelen ser transitorios. El botón de reintentar recupera tu último mensaje para que lo reenvíes con un clic.

| Problema detectado | Mensaje |
|---|---|
| Límite de solicitudes | Se ha superado el límite de solicitudes. Espera un momento e inténtalo de nuevo. |
| Timeout | La solicitud tardó demasiado. Inténtalo de nuevo. |
| Error de red | Error de conexión con el proveedor de IA. Comprueba tu conexión a internet. |
| Error del servidor | Error del servidor de IA. Inténtalo de nuevo en unos segundos. |
| Agente ocupado | El agente está ocupado con otra tarea. Espera a que termine. |
| Sesión perdida | No se pudo crear la sesión del agente. Inténtalo de nuevo. |
| Respuesta vacía | La IA no generó ninguna respuesta. Inténtalo de nuevo. |
| Error del proveedor | El proveedor de IA devolvió un error. Inténtalo de nuevo. |
| Error interno | Error interno de la aplicación. Reinicia Vibes para resolverlo. |

## Proveedores de IA unificados

Las secciones de configuración de OpenRouter, proveedores personalizados y Ollama estaban repartidas por distintas zonas de Ajustes, cada una con su propio diseño y comportamiento. En esta versión se han consolidado en una única card **Proveedores de IA** dentro de Ajustes, con un layout homogéneo para todos: cabecera expandible, indicador de estado y controles de activación/desactivación individuales.

Todos los proveedores configurados funcionan ahora **en paralelo**. Ya no existe el concepto de "proveedor activo" exclusivo: OpenRouter, los custom y Ollama pueden estar habilitados simultáneamente, y sus modelos aparecen juntos en el selector del chat agrupados por proveedor. Cada proveedor (excepto OpenRouter, que es el núcleo y está siempre activo) puede desactivarse individualmente desde su toggle. Al desactivar un proveedor, sus modelos desaparecen instantáneamente del selector de modelo en todo Vibes.

## Soporte completo de Ollama como modelo principal

Ollama ya no está limitado a tareas secundarias. Ahora puedes seleccionar cualquier modelo local de Ollama como **modelo principal del chat**, igual que harías con cualquier modelo de OpenRouter. Al elegirlo en el selector, el agente completo (con todas sus herramientas, contexto y capacidades) funciona directamente contra tu servidor Ollama local.

La integración se basa en la API compatible con OpenAI (`/v1/chat/completions`), lo que significa que cualquier modelo que Ollama soporte funciona sin configuración adicional. Los modelos de Ollama aparecen automáticamente en el selector con su nombre y tag (por ejemplo, **Qwen 3.5 (0.8b-32k)**) para distinguir variantes del mismo modelo base.

## Ajuste fino con el Tuner de Inferencia

Hemos sustituido el antiguo selector de "esfuerzo de razonamiento" por un nuevo **Inference Tuner** en la sección de Ajustes. Esta interfaz te permite ajustar dinámicamente los hiperparámetros de los modelos para afinar sus respuestas a tu gusto: Temperatura (creatividad), Top P (diversidad de vocabulario) y Repetition Penalty (para evitar que el modelo se repita en textos largos).

## Node.js portable

Vibes ahora incluye su propio runtime de Node.js portable empaquetado directamente en la aplicación. Esto significa que ya no necesitas tener Node instalado en tu sistema operativo para usar el agente. Además, se ha integrado un script de compilación automatizado que abre la puerta a la distribución nativa y oficial en Linux.

## Interfaz unificada: El fin de la pantalla de inicio

Para hacer la experiencia más rápida y directa, hemos eliminado por completo la pestaña antigua de "Aplicaciones" (Home). Ahora la aplicación entra directa al Workspace (el entorno del agente de IA), fusionando la gestión de proyectos en la propia barra lateral izquierda. Desde ahí puedes cambiar de proyecto, buscar o crear aplicaciones nuevas sin salir del entorno de trabajo.

Además, las conexiones con servicios externos (GitHub, Bunny, Supabase y PocketBase) ya no están sueltas. Las hemos centralizado en un único **Panel de Integraciones** al que puedes acceder fácilmente desde las opciones del proyecto activo en el menú lateral.

## Gestión integral de Prompts y Categorías

El sistema interno de gestión de prompts ha sido reconstruido desde cero para apoyarse de forma robusta en la base de datos. Desde la sección de Prompts ahora tienes control total: puedes crear tus propias categorías personalizadas, editar sus nombres y descripciones de forma fluida sin salir del menú, y reorganizar tus prompts moviéndolos libremente entre categorías usando el nuevo selector desplegable. Toda la interfaz se ha refinado y cohesionado con el diseño nativo de Vibes, implementando cuadros de diálogo confirmatorios elegantes y un sistema de autoguardado asíncrono e instantáneo (*fire & forget*) al activar o desactivar un prompt. Como parte de esta limpieza, el experimental "Modo Hombre de las Cavernas" ha sido purgado por completo.

## Redacción inteligente con el Modelo Estratega

Ahora puedes utilizar el nuevo **Modelo Estratega** para ayudarte a redactar o perfeccionar tanto las instrucciones de comportamiento (Skills) como las directivas del sistema (Prompts). Desde el propio editor de Prompts o de Skills, dispones de un botón para invocar al asistente: describe en lenguaje natural qué quieres conseguir o qué ajustes deseas realizar (por ejemplo, *«haz que el tono sea más formal y estructurado»*) y el asistente generará una propuesta optimizada que podrás aceptar, refinar o descartar con un solo clic.

## Control total de Skills y gestión por ámbitos

Vibes ahora incluye una nueva sección dedicada en Ajustes para gestionar los **Skills**. Desde esta interfaz renovada puedes configurar y añadir directivas de comportamiento personalizadas para el agente, con la posibilidad de elegir su **ámbito de aplicación** (Scope): puedes crear un skill de forma **Global** (aplicable a todos tus proyectos) o asignarlo específicamente a un **Proyecto** concreto (se almacenará en la carpeta `.opencode/skills` de esa aplicación). La interfaz ahora agrupa visualmente los skills según su ámbito y gestiona automáticamente las operaciones de borrado y renombrado correspondientes.

## Directivas e instrucciones para Servidores MCP

Hemos extendido la configuración de tus herramientas externas. A partir de ahora, cada servidor MCP registrado admite directivas contextuales a través del campo `instructions`. Estas instrucciones te permiten guiar el comportamiento y uso de las herramientas del servidor directamente desde los ajustes de la integración, facilitando que el agente interactúe con ellas de forma mucho más precisa.

## Correcciones y ajustes

- El prompt de generación de títulos de chats ahora es editable desde **Settings → Prompts → Generación de Nombres y Títulos**
- Corregido un bug donde al crear una nueva app desde Home con modo **Plan** seleccionado, el chat se iniciaba en modo Agent ignorando la elección del usuario
- Los artefactos (planes) ahora se re-habilitan para revisión automáticamente cuando el agente modifica el archivo después de haber sido aceptados. Antes, el botón quedaba en "Aceptado" permanentemente sin posibilidad de añadir nuevos comentarios ni volver a enviar la revisión
- Corregido un bloqueo donde las memorias dejaban de generarse tras acumular memorias inactivas. La compactación ahora elimina físicamente las originales en lugar de desactivarlas, la telemetría se ejecuta después de persistir las memorias, y se eliminó la auto-compactación concurrente que podía causar conflictos de escritura. Al arrancar, se purgan automáticamente logs y memorias inactivas de más de 7 días
- Corregida una race condition donde las preferencias de la base de datos no estaban disponibles al entrar a Home, causando un flash de los banners de configuración inicial y que el saldo de OpenRouter no se mostrara hasta refrescar manualmente los ajustes
- Aumentado el timeout de arranque del servidor OpenCode de 5 a 15 segundos, con reintento automático en caso de timeout. Soluciona el error "Timeout waiting for server to start after 5000ms" que se producía en macOS por las verificaciones de Gatekeeper/code signing en el primer arranque
- Optimizado el guardado de ajustes para que los cambios (modelo, tema, permisos) se apliquen de forma instantánea, eliminando el lag perceptible causado por la sincronización síncrona con la base de datos remota
- Los nombres de modelos de Ollama en el selector ahora incluyen el **tag completo** (ej: `0.8b-32k`) para distinguir variantes del mismo modelo base
- La sección **Proveedores de IA** en el sidebar de Ajustes sustituye la antigua entrada "OpenRouter"
- El sistema de ajustes migra internamente de un archivo JSON monolítico a un **almacén clave-valor** con caché en memoria. Las lecturas son instantáneas tras el login, las escrituras se sincronizan entre ventanas en tiempo real y la persistencia es asíncrona. El cambio es transparente: la experiencia de uso no varía
- El catálogo de modelos de OpenRouter se ha **limpiado a fondo**: se filtran más de 70 modelos irrelevantes para desarrollo (legacy, demasiado pequeños, de roleplay o duplicados) y se reactivan las variantes gratuitas (`:free`)
- El autoguardado asíncrono (*fire & forget*) aplicado recientemente a las Preferencias Globales se ha extendido también a la creación, edición y activación de prompts y categorías, haciendo la interfaz mucho más ágil al eliminar los bloqueos de red
- La plantilla de **React 19** se ha integrado de forma nativa y ahora es la opción por defecto al crear nuevos proyectos (sustituyendo a la antigua `react-beta`)
- Limitado el panel de visualización del razonamiento en vivo (Live Thinking) a los últimos 10,000 caracteres para evitar problemas de rendimiento y lentitud visual cuando el agente genera razonamientos extremadamente largos
- Si el proceso de renderizado visual de Vibes sufre un error crítico (crash, falta de memoria o fallo de integridad), la aplicación ya no se queda en blanco; ahora muestra un aviso explicando la situación y ofrece un botón para recargar la ventana y recuperar el estado de inmediato sin reiniciar toda la aplicación
