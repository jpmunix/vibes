# v8.6
*13 de mayo de 2026*

## Tareas que no se pierden

Hasta ahora, si recargabas la página o cerrabas la ventana mientras el agente estaba trabajando, al volver encontrabas una burbuja vacía sin forma de saber qué había pasado. A partir de esta versión, cada tarea del agente se guarda en tiempo real: el contenido se va almacenando cada pocos segundos, y el estado de la tarea (en progreso, completada, fallida) queda registrado de forma permanente. Si vuelves a abrir el chat, Vibes detecta si la tarea sigue activa y actualiza el contenido automáticamente, o te muestra directamente el resultado final si ya terminó. Nada se pierde, aunque cierres y te vayas a tomar un café.

## Errores que se explican solos

Cuando algo falla en la comunicación con la IA, ya no verás un bloque de JSON técnico incomprensible. Vibes ahora intercepta cada error, lo clasifica internamente y te muestra un mensaje claro en español explicando qué ha pasado. Pero la mejora no se queda ahí: dependiendo del tipo de error, la burbuja incluye **botones de acción directa** para resolver el problema sin salir del chat.

Si el error es algo que se puede solucionar reintentando (un timeout, un error puntual del servidor, un problema de red), verás un botón **Reintentar** que recupera tu último mensaje y lo prepara para reenviar. Si el error requiere que hagas algo distinto (cambiar de modelo, recargar créditos, revisar tu API key), los botones te llevan directamente a donde necesitas ir: a Ajustes, a la página de créditos de OpenRouter o a un chat nuevo. En todos los casos puedes desplegar la sección **Ver detalles** para consultar el error técnico original si necesitas diagnosticar algo en profundidad.

Antes de mostrar cualquier error, Vibes intenta reparar el problema de forma silenciosa. Si lo consigue, la conversación continúa sin interrupciones y sin que notes nada. Si el auto-repair no funciona, entonces sí se muestra el error con las opciones manuales.

| Situación | Qué hace Vibes |
|---|---|
| El servidor del agente se ha caído | Lo reinicia automáticamente y reintenta tu mensaje |
| La sesión del agente se ha perdido | Crea una sesión nueva y reintenta |
| El agente está ocupado con otra tarea | Aborta la tarea anterior, espera 2 segundos y reintenta |
| Límite de solicitudes alcanzado | Espera 5 segundos y reintenta |

<!-- @tip "El auto-repair se ejecuta como máximo una vez por mensaje. Si el segundo intento también falla, se muestra el error al usuario con las opciones de acción manual." -->

### Errores irrecuperables

Estos errores requieren una acción del usuario para resolverse. El botón de reintentar no aparece porque enviar el mismo mensaje no solucionaría nada.

| Problema detectado | Mensaje | Acciones disponibles |
|---|---|---|
| Créditos agotados | Parece que se agotaron los créditos de IA de tu cuenta. | Recargar créditos, Cambiar modelo |
| API key inválida | Parece que hay un problema con tu clave API. Revísala en ajustes. | Abrir Ajustes |
| Modelo no disponible | Parece que el modelo seleccionado no está disponible. Prueba con otro. | Cambiar modelo |
| Chat demasiado largo | Parece que el chat es demasiado largo para el modelo. Abre un nuevo chat o cambia a un modelo con mayor ventana de contexto. | Nuevo chat, Cambiar modelo |
| Contenido bloqueado | Parece que el contenido fue bloqueado por los filtros de seguridad del modelo. | — |
| Agente no encontrado | Parece que no se encontró el agente de IA. Reinicia Vibes para resolverlo. | — |
| Disco lleno | Parece que no queda espacio en disco. Libera espacio e inténtalo de nuevo. | — |

### Errores recuperables

Estos errores suelen ser transitorios. El botón de reintentar recupera tu último mensaje para que lo reenvíes con un clic.

| Problema detectado | Mensaje |
|---|---|
| Límite de solicitudes | Se ha superado el límite de solicitudes. Espera un momento e inténtalo de nuevo. |
| Timeout | La solicitud tardó demasiado. Inténtalo de nuevo. |
| Error de red | Error de conexión con el proveedor de IA. Comprueba tu conexión a internet. |
| Error del servidor | Error del servidor de IA. Inténtalo de nuevo en unos segundos. |
| Agente ocupado | El agente está ocupado con otra tarea. Espera a que termine. |
| Sesión perdida | No se pudo crear la sesión del agente. Inténtalo de nuevo. |
| Respuesta vacía | La IA no generó ninguna respuesta. Inténtalo de nuevo. |
| Error del proveedor | El proveedor de IA devolvió un error. Inténtalo de nuevo. |
| Error interno | Error interno de la aplicación. Reinicia Vibes para resolverlo. |

## Correcciones y ajustes

- El prompt de generación de títulos de chats ahora es editable desde **Settings → Prompts → Generación de Nombres y Títulos**
- Corregido un bug donde al crear una nueva app desde Home con modo **Plan** seleccionado, el chat se iniciaba en modo Agent ignorando la elección del usuario
- Los artefactos (planes) ahora se re-habilitan para revisión automáticamente cuando el agente modifica el archivo después de haber sido aceptados. Antes, el botón quedaba en "Aceptado" permanentemente sin posibilidad de añadir nuevos comentarios ni volver a enviar la revisión
- Corregido un bloqueo donde las memorias dejaban de generarse tras acumular memorias inactivas. La compactación ahora elimina físicamente las originales en lugar de desactivarlas, la telemetría se ejecuta después de persistir las memorias, y se eliminó la auto-compactación concurrente que podía causar conflictos de escritura. Al arrancar, se purgan automáticamente logs y memorias inactivas de más de 7 días
- Corregida una race condition donde las preferencias de la base de datos no estaban disponibles al entrar a Home, causando un flash de los banners de configuración inicial y que el saldo de OpenRouter no se mostrara hasta refrescar manualmente los ajustes
- Aumentado el timeout de arranque del servidor OpenCode de 5 a 15 segundos, con reintento automático en caso de timeout. Soluciona el error "Timeout waiting for server to start after 5000ms" que se producía en macOS por las verificaciones de Gatekeeper/code signing en el primer arranque
- Optimizado el guardado de ajustes para que los cambios (modelo, tema, permisos) se apliquen de forma instantánea, eliminando el lag perceptible causado por la sincronización síncrona con la base de datos remota
